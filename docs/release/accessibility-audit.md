# Accessibility Verification

## Screen Reader Walkthrough

### iOS VoiceOver
- Journal: open palette, entry context menu, conflict review modal.
- Habits: open palette, context actions, closeout elements.
- Calendar: open palette, day card controls, reflection input.
- Verify control names are announced and focus order is logical.

### Android TalkBack
- Repeat same core flows as VoiceOver.
- Ensure touch exploration reaches all actionable controls.
- Ensure modal boundaries are respected (no background traversal).

## Keyboard Audit
- Tab/Shift+Tab order remains within open dialog.
- Escape closes overlays consistently.
- Enter activates selected command/action buttons.

## Contrast and Readability
- Verify text/icon contrast in light and night modes.
- Validate status badges and warning/error toasts meet contrast.
- Confirm focus ring visibility on keyboard navigation.

## Live Region Audit
- Confirm sync/error/undo announcements are spoken.
- Confirm announcements are concise and not duplicated.
